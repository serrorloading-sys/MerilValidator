⚙️ Matching Logic Passes
Exact MatchAll three — Material, Batch, and Serial — must match exactly with SAP records. This is the strictest and most accurate match type. Always ON and cannot be disabled.
Concat MatchSome barcodes encode the Serial number concatenated with the Batch (e.g. B001S001). When enabled, the validator automatically tries Batch+Serial combined as the serial key against SAP records.
Batch MatchIf no exact match is found, the validator tries matching on Material + Batch only, ignoring the Serial number. Result shows "Partial" status.
Material MatchThe loosest match — only the Material code is compared; Batch and Serial are ignored. Results show "Warning" status. Turn OFF for strict serial-level validation.
🔧 Consumption Improvements
Smart SortWhen multiple SAP rows are available for the same Material/Batch, the best row is selected using a three-level sort: 1) Plant Priority list → 2) Nearest expiry date → 3) Highest available quantity. When OFF, only Plant Priority sorting applies.
Multi-Row Qty SplitIf a scan's quantity exceeds what one SAP row can provide, the remaining quantity is consumed from the next best SAP row. When OFF, only one row is consumed — remaining qty shows as "Variance (Partial)".
🧪 Advanced Validation Logic
Duplicate ScanFlags rows where the same barcode (Material+Batch+Serial) appears more than once in the scan file. Detects scanning errors or accidental double-entries.
Serial ConflictFlags when the same serial number appears with different Material or Batch combinations in the scan file. Critical error — serial numbers must be globally unique per item.
Over-Scan AlertFlags rows where the total scanned quantity for a batch exceeds the available SAP quantity (e.g. SAP has 5 but you scanned 7).
Plant MismatchWarns when a matched SAP item belongs to a plant not listed in your Plant Priority settings. Fill in your priority plants (e.g. WGJ1, WGJ7) in Settings.
Incomplete BarcodeFlags barcodes where the Batch or Serial AI code is missing. Detects poor barcode print quality or unsupported barcode formats.
Expiry MismatchCompares the GS1 Application Identifier (17) expiry date embedded in the barcode against the SAP SLED date. Flags if the difference exceeds 7 days. Detects labelling errors or wrong batch assignments.
Qty Recon KPIAdds an extra KPI card on the dashboard showing the percentage match between total SAP quantity and total scanned quantity (e.g. "94.5% — 12 items unscanned").

About This Tool

What is this?

The SAP Inventory Validator is a client-side tool designed to assist Warehouse Management teams in reconciling physical stock counts (scanned via GS1 barcodes) against SAP system dumps.

How it Works

The tool uses a Waterfall Matching Logic to link scanned items with SAP records:

  1. Exact Match: Matches Material + Batch + Serial. (Highest Confidence)
  2. Batch Match: Matches Material + Batch (if Serial differs or is missing).
  3. Material Match: Matches Material only (if Batch/Serial differ).

Logic & Rules

  • Stock Status: Determined by Expiry Date and "System Condition".
  • Customer Lookup: Fetches Customer Name from Master Data based on Customer Code (supports fuzzy string matching).
  • Storage Location: Automatically assigned based on Status (Good Stock vs Damage) using your configured codes.
Privacy Note: This tool runs 100% in your browser. No data is uploaded to any server. Your files remain on your device.
